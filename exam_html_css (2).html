PARTIE 1 – QUESTIONS THÉORIQUES (30 points)

1. Structure d'une page HTML 6 pts

1.1. Structure minimale HTML5 valide
<!DOCTYPE html> <html lang="fr"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <title>Titre de la page</title> </head> <body> <!-- Contenu visible --> </body> </html>
1.2. Rôles des balises principales
  • <!DOCTYPE html> : Déclare que le document est en HTML5 et permet au navigateur d'interpréter correctement la page.
  • <html> : Balise racine qui encadre tout le document HTML et définit la langue avec l'attribut lang.
  • <head> : Section contenant les métadonnées (titre, encodage, style, scripts) invisibles à l'écran mais essentielles pour le navigateur.
  • <body> : Section contenant tout le contenu visible de la page (texte, images, liens, etc.).

2. Balises et attributs 6 pts

2.1. Différence balise vs attribut
  • Balise HTML: C'est un élément structurant du document qui encadre le contenu avec une paire de symboles < et >. Elle définit la nature du contenu (par exemple, <p> pour un paragraphe, <h1> pour un titre).
  • Attribut HTML: C'est une propriété ou une caractéristique ajoutée à l'intérieur d'une balise pour modifier son comportement ou ses propriétés. Il se présente sous la forme nom="valeur" (par exemple, href="lien.html").
2.2. Exemple de balise avec deux attributs
<a href="https://example.com" title="Cliquez ici">Lien vers le site</a>
  • href: Cet attribut spécifie l'adresse URL de destination vers laquelle le lien renvoie.
  • title: Cet attribut affiche un texte informatif au survol du lien pour améliorer l'accessibilité et l'expérience utilisateur.

3. Sémantique HTML 6 pts

Balise Rôle Exemple
<div> Conteneur générique sans signification sémantique, utilisé uniquement pour regrouper et organiser le contenu. <div class="conteneur"><p>Texte</p></div>
<header> Représente l'en-tête d'une page ou d'une section, contenant généralement le logo, le titre principal et la navigation. <header><h1>Mon site</h1></header>
<nav> Encadre les liens de navigation principaux, indiquant aux moteurs de recherche qu'il s'agit de navigation. <nav><a href="/">Accueil</a></nav>
<section> Regroupe du contenu thématiquement lié formant une section logique de la page avec généralement un titre. <section><h2>À propos</h2></section>
<article> Encadre un contenu autonome et réutilisable (billet de blog, article) qui pourrait être distribué indépendamment. <article><h3>Titre</h3></article>

4. Sélecteurs CSS 6 pts

Sélecteur de type: Sélectionne tous les éléments d'un type HTML spécifique en utilisant le nom de la balise.
p { color: black; font-size: 16px; }
Sélecteur de classe: Sélectionne tous les éléments ayant une classe spécifique en utilisant un point (.) suivi du nom de la classe.
.important { font-weight: bold; color: red; }
Sélecteur d'identifiant: Sélectionne un seul élément portant un identifiant unique en utilisant un dièse (#).
#menu { background-color: navy; padding: 20px; }

5. Box model et types d'éléments 6 pts

5.1. Les quatre parties du CSS box model:
  1. Contenu (Content): La zone contenant le texte ou les images, dimensionnée par width et height.
  2. Remplissage (Padding): L'espace blanc à l'intérieur de la boîte, entre le contenu et la bordure.
  3. Bordure (Border): La ligne autour du padding, définie par la propriété border.
  4. Marge (Margin): L'espace transparent à l'extérieur de la bordure, séparant cet élément des autres.
5.2. Différence block vs inline:
  • Block: Les éléments bloc occupent toute la largeur disponible et créent un retour à la ligne avant et après eux. Exemples: <div>, <p>, <h1>, <section>
  • Inline: Les éléments inline ne prennent que l'espace nécessaire et restent sur la même ligne que les autres contenus. Exemples: <span>, <a>, <strong>, <em>

PARTIE 2 – ANALYSE DE CODE HTML/CSS (30 points)

6. Analyse d'un extrait HTML 15 pts

6.1. Balises sémantiques (4 pts)
  • <h1> : Titre principal de la page
  • <p> : Paragraphes de texte
  • <footer> : Pied de page
6.2. Différence class="bloc" vs class="bloc important" (4 pts)

L'élément avec class="bloc" a UNE seule classe. L'élément avec class="bloc important" a DEUX classes appliquées simultanément.

6.3. Amélioration sémantique (3 pts)

Proposition: Remplacer <div class="bloc"> par <article>

6.4. Balise manquante (4 pts)
<link rel="stylesheet" href="style.css">

7. Analyse d'un extrait CSS 15 pts

7.1. Différence h1.titre vs .bloc.important (4 pts)

h1.titre: Cible seulement les <h1> avec classe "titre"
.bloc.important: Cible les éléments avec SIMULTANÉMENT les deux classes

7.2. Apparence visuelle (4 pts)
  • Bordure: 2px solid #333
  • Padding: 15px
  • Margin: 10px
  • Arrière-plan: #f5f5f5
7.3. Effet de #intro { color: red; } (3 pts)

Le texte devient rouge. L'id a une spécificité très élevée.

7.4. Règle CSS pour footer (4 pts)
footer { text-align: center; font-size: 14px; background-color: #f0f0f0; }

9. Questions Responsive 10 pts

9.1. Responsive Design (5 pts)

Le responsive design est une approche de conception web qui rend les sites adaptables à tous les appareils et tailles d'écran. Important car 60% du trafic vient du mobile et Google privilégie les sites mobile-friendly.

9.2. Media query ≤ 600px (5 pts)
@media (max-width: 600px) { body { font-size: 14px; } header h1 { font-size: 1.8em; } #navigation { flex-direction: column; gap: 10px; } .projet { padding: 15px; margin: 10px 0; } }
